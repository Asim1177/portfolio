What is Full Stack Development?

Full Stack Development means building both the client-facing side of a web application (the front-end) and the server-side logic and data management (the back-end). A Full Stack Developer understands the complete journey of a web request — from a user clicking a button, to a server processing data, to a database storing the result.

Common stacks include the MERN stack (MongoDB, Express, React, Node.js) and the LAMP stack (Linux, Apache, MySQL, PHP). I work primarily with MERN.

Back-End Example (Node.js + Express)

A minimal REST API route that returns a JSON response:

const express = require('express');
const app = express();

app.get('/api/developer', (req, res) => {
  res.json({
    name: "Asim Titung",
    role: "Full Stack Developer",
    stack: ["React", "Node.js", "MongoDB"]
  });
});

app.listen(3000, () => {
  console.log('API running at http://localhost:3000');
});

Output (JSON response):

{
  "name": "Asim Titung",
  "role": "Full Stack Developer",
  "stack": ["React", "Node.js", "MongoDB"]
}

Database Example (SQL)

Creating and querying a projects table using SQL:

CREATE TABLE Projects (
  id       INT PRIMARY KEY AUTO_INCREMENT,
  title    VARCHAR(100),
  tech     VARCHAR(100),
  status   VARCHAR(20)
);

INSERT INTO Projects (title, tech, status)
VALUES
  ('Analytics Dashboard', 'React, MongoDB', 'Live'),
  ('Study Planner',       'Vanilla JS',     'Live'),
  ('Recipe API',          'Express, PostgreSQL', 'In Progress');

SELECT * FROM Projects;
